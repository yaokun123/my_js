传统的开发会有全局变量命名冲突等问题

可以用闭包来解决命名冲突的问题（函数有自己的作用域）: 

    (function(){
        var flag = true;
    })()

但是如果引入闭包的话代码的复用性不高

如果想解决命名冲突问题并且让代码复用性较高就可以使用模块化

用闭包实现模块化

    var moduleA = (function(){
        var obj = {};

        var flag = true;

        //导出属性
        obj.flag = flag;
        return obj;
    })()

常见的模块化规范

1、CommonJS(node.js就是实现这个规范)

    //CommonJS的导出
    module.exports = {
        flag:true,
        test(a,b){
            return a + b;
        }
    };

    //CommonJS的导入
    var {flag,sum} = require('xxx.js')
